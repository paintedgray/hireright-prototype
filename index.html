HireRight prototype

An AI-augmented hiring workflow — built with bias control upstream, not bolted on.

A working prototype for a recruiter triaging 150 applicants on a senior-engineering role. Three screens demonstrate the thesis: define what "good" looks like before the model produces a shortlist, show confidence bands honestly inside the commitment, and capture override reasoning as first-class data.

Walkthrough
1
Criteria Builder + Proxy Stress Test
Define must-haves, weighted preferences, and signal tuning. The system runs a pre-flight bias check on the criteria themselves — a three-tier action system (soft flag · soft block · hard block).
→
2
Ranked Candidate View + Confidence Bands Start the demo flow here
The recruiter's working surface. 150 candidates grouped into Tier A (high confidence), Tier B (statistically indistinguishable), and Tier C (mixed signals). All candidates remain visible — no auto-rejection.
→
3
Structured Override Capture Hero
When the recruiter promotes, demotes, or dismisses a candidate, capture why as structured data — not free text. Reason category → auto-suggested detail → optional notes. Pattern detection runs quietly in the background.
→
4
Per-Candidate Context Drawer
Evidence-based context — not score numbers. Auto-extracted snapshot, must-have checklist with green ✓ / amber ⚠ markers, and a resume excerpt with the matching bullets highlighted. Sticky override actions at the bottom.
→
5
Decision Log Compliance
Audit-grade decision trail. Every override, criteria change, and flag acknowledgment captured with timestamp, human attribution, and structured reason. Filter, search, and export to CSV for compliance handoff.
→
Demo flow (~2 min): Open Screen 2 → scroll past the shortlist commit line → click Candidate 12 at rank #15 → drawer (Screen 4) shows strong must-have evidence with only AI/ML missing (a Preferred) → click ↑ Promote to top 10 → structured override capture (Screen 3) opens with the spec-pre-filled reason → Submit → return to Screen 2 and watch Candidate 12 land in Tier A with an audit-tagged badge. Then click 📋 Decision log to see the override land in the audit trail alongside historical entries. Hit ↻ Reset demo on Screen 2 to start clean.
